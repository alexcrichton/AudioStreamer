iOSStreamerDelegate Protocol Reference

Conforms to AudioStreamerDelegate
Declared in iOSStreamer.h

Overview

The iOSStreamerDelegate protocol extends the AudioStreamerDelegate and provides callbacks for the interruption events that the iOSStreamer class deals with. You can use these callbacks to override the default behaviour on interruption.

– streamerInterruptionDidBegin:

Called when the stream is interrupted

- (BOOL)streamerInterruptionDidBegin:(iOSStreamer *)sender

Parameters

sender

The streamer that called this method

Return Value

YES to override the default behaviour of pausing the stream

iOSStreamer.h

– streamer:interruptionDidEndWithFlags:

Called when the stream interruption has ended

- (BOOL)streamer:(iOSStreamer *)sender interruptionDidEndWithFlags:(AVAudioSessionInterruptionOptions)flags

Parameters

sender

The streamer that called this method
flags

The flags received indicating whether the stream should resume. This will be AVAudioSessionInterruptionOptionShouldResume if it should resume or 0 if it should not.

Return Value

YES to override the default behaviour of playing the stream if the flag allows and stopping if it does not
